### 1.2 医院数据源调研（关键风险点）

**任务**：
- [ ] **调研目标医院**（选择3-5家）
  - 大型三甲医院（如北京协和、上海瑞金）
  - 地方医院（如深圳市人民医院）
  - 优先选择有公开API或规范化页面的医院
  
- [ ] **逐家分析**：
  - 挂号系统类型（自建/第三方平台）
  - 号源数据获取方式
    - ✅ 优先：官方API
    - ⚠️ 次选：规范HTML页面爬取
    - ❌ 避免：反爬虫严格的系统
  - 反爬虫机制（User-Agent、验证码、IP限制）
  - 数据更新频率
  
- [ ] **建立医院库**
  ```json
  {
    "hospitals": [
      {
        "id": "hospital_001",

        "api_endpoint": "https://...",
        "crawl_strategy": "api",

  ```

- [ ] **出行/地图平台调研**
  - 高德地图API（距离、时间）
  - 12306/携程API（交通方案）
  - 酒店平台（住宿价格）

## ⚠️ 风险管理

### 高风险项

| 风险 | 影响 | 应对方案 |
|------|------|--------|
| 医院数据采集失败 | 阻塞整个项目 | 第1周完成调研，立即切换备选医院 |
| 反爬虫机制升级 | 数据采集中断 | 准备模拟数据方案，支持离线演示 |
| 演示环境网络问题 | 答辩失败 | 预录演示视频作为备份 |
| 时间不足 | 功能不完整 | 优先完成场景A，场景B/C可简化 |

### 应急预案

- **如果医院数据采集困难**：
  - 使用模拟数据完成演示
  - 强调Skill的可复用性和架构设计
  
- **如果时间紧张**：
  - 优先级：场景A > 场景C > 场景B
  - 可以只实现2个完整场景
