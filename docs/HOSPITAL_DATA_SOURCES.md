# 北京医院数据源调研

## 1. 第三方挂号平台

### 1.1 114挂号网
- **官网**: https://www.114ygk.com/
- **特点**: 北京最大的挂号平台，覆盖大多数三甲医院
- **数据**: 医院、科室、医生、号源、费用
- **API**: 有非官方API，但需要反向工程
- **可行性**: ⭐⭐⭐⭐ (高)

### 1.2 京医通
- **官网**: https://www.bjguahao.gov.cn/
- **特点**: 北京市政府官方挂号平台
- **数据**: 医院、科室、医生、号源、费用
- **API**: 官方API（需要申请）
- **可行性**: ⭐⭐⭐⭐⭐ (最高，官方支持)

### 1.3 微医
- **官网**: https://www.guahao.com/
- **特点**: 全国性挂号平台
- **数据**: 医院、科室、医生、号源、费用
- **API**: 有API接口
- **可行性**: ⭐⭐⭐⭐ (高)

## 2. 医院官方渠道

### 2.1 北京协和医院
- **官网**: https://www.pumch.cn/
- **挂号**: https://www.pumch.cn/
- **API**: 无公开API
- **爬虫**: 可行（网站结构相对稳定）
- **可行性**: ⭐⭐⭐ (中)

### 2.2 北京大学第一医院
- **官网**: https://www.bjmu.edu.cn/
- **挂号**: 通过114或京医通
- **API**: 无公开API
- **可行性**: ⭐⭐⭐ (中)

### 2.3 北京市第一中心医院
- **官网**: https://www.bjszyy.com.cn/
- **挂号**: 通过114或京医通
- **API**: 无公开API
- **可行性**: ⭐⭐⭐ (中)

## 3. 推荐方案

### 优先级顺序：
1. **京医通API** (官方、最稳定)
2. **114挂号网爬虫** (覆盖面广、数据全)
3. **医院官网爬虫** (备选)
4. **模拟数据** (降级方案)

## 5. 技术栈

- **HTTP请求**: requests / httpx
- **浏览器自动化**: Playwright / Selenium
- **数据解析**: BeautifulSoup / lxml
- **缓存**: Redis / SQLite
- **日志**: logging

## 6. 风险和应对

| 风险 | 应对方案 |
|------|--------|
| API限流 | 实现缓存、请求队列 |
| 反爬虫 | 使用代理、随机延迟、User-Agent轮换 |
| 网站变更 | 定期维护、监控爬虫失败 |
| 数据不一致 | 数据验证、多源对比 |
