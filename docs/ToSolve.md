# 🤖 智能体自动化协作专项赛官方指导手册

## 🌟 赛项核心主旨

信息技术应用创新已从底层硬件延伸至上层人工智能应用。本专项赛旨在基于 **智谱 AI GLM 大模型底座 + 清流智能体开发平台 + AutoClaw 智能调度框架**，培养学生构建能够自主处理复杂任务的“数字员工”能力，实现从“对话”到“执行”的跨越。

## 🏆 一、 赛题方向与内容（三选一）

本赛项下设三个赛题方向，参赛团队可选择其中一个进行创作：

### （一）赛题一：校园服务智能体——实现“零跑腿”的自动化服务

- **功能描述：** 围绕学生日常生活需求进行智能体搭建。鼓励从“咨询”转向“行动”。

- **典型场景：** 自动报修助手（拍照后自动识别故障类型并生成标准化报修工单）、失物招领自动匹配官（自动抓取公告信息并私信通知失主）、一键证明办理助手（基于公开接口完成证明申请流程）。

> **📖 官方案例 1：教务公文“速阅”数字助手**
> 
> - **真实痛点：** 学校中层领导或行政老师每天需处理大量冗长的公文，很难在短时间内抓取核心指令（谁负责、何时截止、交什么材料）。
> 
> - **核心 Skill 封装：**
>   
>   - *结构化要素提取 Skill：* 自动识别文号、日期、部门。
>   
>   - *领导视角摘要 Skill：* 生成 100-300 字精简摘要，突出“核心任务”和“动作建议”。
> 
> - **自动化闭环：**
>   
>   - *指令输入：* “帮我梳理本周教务处的 5 份公文。”
>   
>   - *动作产出：* AutoClaw 读取文档、解析摘要，自动生成“公文简报”并可通过 AutoClaw 发送给相关负责人。

### （二）赛题二：学习成长智能体——构建“学术与职场”数字助手

- **功能描述：** 围绕学生学习生活与职业发展进行智能体搭建。

- **典型场景：** 全自动文献调研员（自动搜索、下载并提取关键数据）、简历自动优化与投递闭环智能体。

> **📖 官方案例 2：个性化校招简历生成助手**
> 
> - **真实痛点：** 毕业生面对空白模板无从下手，不知如何将凌乱的校园经历转化为职场规范描述。
> 
> - **自动化闭环：**
>   
>   - *指令输入：* “这是我大学经历草稿和证书，帮我生成针对‘管培生’的简历。”
>   
>   - *动作产出：* AutoClaw 识别照片、提取经历，GLM 依 STAR 原则打磨，自动导出排版精美的 Word/PDF 简历。

> **📖 官方案例 3：行业实习职位自动聚合器**
> 
> - **自动化闭环：**
>   
>   - *指令输入：* “帮我找上海的 AI 产品实习。”
>   
>   - *动作产出：* AutoClaw 跨平台翻页抓取岗位数据，GLM 根据简历打分，生成《实习申请对比表.xlsx》。

### （三）赛题三：通用社会创新智能体（开放类）——跨应用自动化协作

- **功能描述：** 不设固定场景，鼓励实现跨应用（如浏览器到 Excel 再到邮件）的长链路自动化协作。

- **典型场景：** 全自动个人旅行管家、个人健康数据自动采集顾问。

> **📖 官方案例 4：旅行规划助手**
> 
> - **真实痛点：** 制定攻略需反复切换小红书、携程、高德，存在大量机械“复制粘贴”。
> 
> - **自动化闭环：**
>   
>   - *指令输入：* “想去大理玩三天，预算三千，帮我把小红书攻略景点串起来，做个 Excel 行程表。”
>   
>   - *动作产出：* GLM 萃取攻略并规划路径，AutoClaw 查价格，最终本地生成含时间轴与费用的《路书.xlsx》。

## ⚠️ 二、 赛项硬性要求

1. **原创性：** 作品必须是 **2026年3月1日至2026年6月30日** 期间完成的原创作品。

2. **技术限制：** 必须使用 **清流平台** 构建 + **AutoClaw** 执行，选用 **GLM** 系列模型。

3. **团队构成：** 同一所院校的 **2—9 名本科生**组成，指导教师不多于 2 人。

4. **作品发布：** 必须具备完善的头像、功能介绍，并发布至智能体商店。

## 🛠️ 三、 比赛形式与技术路径

本赛项旨在培养构建“数字员工”能力，实现**从“对话”到“执行”**的跨越。参赛团队可从以下两种路径中择一创作：

### 📌 路径 A：平台编排模式

- **适用场景：** 逻辑分支复杂、需高频调用知识库的校园服务或咨询类场景。

- **开发方式：** 在清流智能体开发平台中，利用可视化工作流（Workflow）编排业务逻辑（多轮问答、条件判断、知识库检索）。

- **能力集成：** 将编排好的智能体封装为 **MCP (Model Context Protocol)** 协议插件，接入 AutoClaw 框架执行。

- **发布要求：** 须发布至清流智能体商店，具备完善头像和介绍。

### 📌 路径 B：原生 Skill 模式

- **适用场景：** 强调跨应用协作（浏览器->Excel->邮件）、流程高度自动化的社会创新场景。

- **开发方式：** 基于 AutoClaw 框架，直接将核心业务能力（数据采集、信息整理、跨应用写数）封装为**独立的 Skill（技能）**。

- **能力集成：** 通过配置文件定义逻辑，由 GLM 大模型直接调度封装好的 Skill 库完成任务。

- **发布要求：** 在网盘中提交完整的 Skill 文件夹（含 `SKILL.md` 入口文件及辅助文件），详细说明参数及规则。

## 📁 四、 提交材料清单

采用 **“组委会平台注册基本信息 + 百度网盘存储作品材料”** 的模式。

**必交材料：**

1. **作品信息概要表：** 中国大学生计算机设计大赛统一模板。

2. **智能体访问链接（二选一）：**
   
   - *路径 A：* 提交发布至商店的有效访问链接。
   
   - *路径 B：* 提交完整的 Skill 文件夹至百度网盘（以 `SKILL.md` 为入口）。

3. **执行流程图：** 需明确标注“模型思考”与“智能体执行”的具体步骤。

4. **演示视频：** **不超过 10 分钟**。需含开发者视角录屏，展示完整闭环（任务输入->执行过程->最终结果）。**关键执行环节不可过度剪辑**，需连贯展示。

5. **设计文档：** 包含场景痛点分析（拒绝伪命题）、系统架构、Skill 编排逻辑及知识库搭建说明。

## ⚖️ 五、 核心评审标准

评委将严格参考 **“发一句话，结果产出”** 的闭环能力进行打分：

1. **核心能力 Skill 化：** 是否有效将复杂流程拆解并封装为具备原子化能力的 Skill。

2. **端到端闭环：** 实测作品是否能**真正替代人工操作**，完成从指令输入到最终文档/动作产出的完整链路。
