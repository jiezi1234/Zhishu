# 全人群智能就医调度与医旅管家（HealthPath Agent）

## 1. 项目定位与赛题三契合

### 1.1 项目一句话
用户只需一句话描述就医需求，系统自动完成“号源检索—方案比选—跨应用执行—结果文档输出”的端到端闭环。

### 1.2 对应赛题三核心要求（逐条映射）
- **通用社会创新智能体（开放类）**：面向老年人、职场人、异地就医家庭，解决高频民生问题。
- **跨应用自动化协作**：浏览器站点抓取 + 地图/出行平台 + 文档与表格输出 + 消息通知。
- **从对话到执行**：不是问答助手，而是可执行任务的“数字员工”。
- **可复用 Skill 化能力**：将复杂流程拆分为多个原子 Skill 并支持编排。

## 2. 技术路线（按官方要求落地）

### 2.1 硬性技术栈
- 模型：`GLM` 系列
- 执行框架：`AutoClaw`
- 编排平台：`清流平台`

### 2.2 路径选择
采用 **路径 B（原生 Skill 模式）** 为主：
- 核心能力封装为独立 Skill
- 通过配置文件定义调度逻辑
- 由 GLM 直接调度 Skill 执行

> 备选：可同步准备路径 A 的发布版页面，作为展示加分项。

## 3. 核心 Skill 设计

### Skill 1：就医意图理解与约束抽取（GLM）
- 输入：用户自然语言（文本/语音）
- 输出：结构化任务 JSON
- 关键字段：症状、目标城市、时间窗口、预算、出行偏好、是否异地

### Skill 2：跨院号源巡航与标准化（AutoClaw）
- 输入：任务 JSON
- 动作：自动访问医院挂号门户/预约页面并抓取可用号源
- 输出：统一表结构（医院、科室、医生、时间、费用、排队预估）

### Skill 3：医旅协同与多目标决策（GLM + AutoClaw）
- 输入：号源数据 + 用户约束
- 动作：联动出行/住宿平台采集数据，计算综合成本与时效
- 输出：Top-N 可执行方案（默认 Top 2）

### Skill 4：结果生成与触达（AutoClaw）
- 输出物：
  - 老年友好版 PDF《就医行程单》
  - 标准版 Excel《医旅全景路书.xlsx》
- 触达方式：邮件/IM（可选）

## 4. 端到端执行闭环（评审重点）

1. 用户输入一句话任务
2. GLM 完成意图识别与参数结构化
3. AutoClaw 执行跨站点抓取与比价
4. GLM 进行方案评估与排序
5. AutoClaw 自动产出 Excel/PDF 并发送

闭环定义：**“一句话输入 -> 自动执行过程 -> 最终可用结果产出”**。

## 5. 典型场景（用于演示）

### 场景 A：银发族陪诊
- 输入示例：
  - “老人这两天腰疼，帮我找本周可挂上的骨科号，并做一份大字版行程单。”
- 输出：可执行挂号建议 + 大字版 PDF

### 场景 B：职场人夜间/周末就医
- 输入示例：
  - “我在南山区上班，只能周末看颈椎，帮我找最近且排队短的医院。”
- 输出：距离与时长最优方案 + 交通建议

### 场景 C：异地医旅一体化
- 输入示例：
  - “下周从赣州去广州看呼吸科，帮我把挂号、车票、住宿一起规划。”
- 输出：一体化医旅路书 Excel + 行程提醒
