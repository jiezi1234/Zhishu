## 1. 场景痛点分析

### 1.1 真实用户问题

普通用户在就医时面临"三不知"困境：

| 痛点 | 具体表现 |
|------|---------|
| 不知挂什么科 | 症状模糊，不清楚对应哪个专科，挂错号浪费时间 |
| 不知去哪看 | 北京三甲医院众多，不知哪家就近、哪家对症、哪家挂得上 |
| 不知怎么去 | 需要手动查地图、查公交，在多个 App 之间反复切换 |

### 1.2 特殊人群痛点

- **老年人**：不熟悉手机操作，难以独立完成在线选医院→找科室→网上挂号的完整流程
- **职场人**：时间紧，只能利用夜间/周末就医，需要精准匹配号源时间窗口
- **异地患者**：跨城市就医需同时协调挂号、交通、住宿，信息散落在多个平台

### 1.3 现有方案不足

现有挂号 App（京医通、微信挂号等）只解决"挂号"单一环节，用户仍需自己完成：症状自判断 → 选医院 → 选科室 → 规划路线。本 Skill 旨在串联全链路，实现"一句话输入 → 全流程方案输出"。

---

## 2. 产品定位

**HealthPath Agent** 是一个面向普通用户的 AI 驱动智能就医调度助手，基于 MCP 服务与多源医疗/地理数据，提供「从症状到就医」的全链路决策支持。

```
用户输入：当前地址 + 病情/症状描述
    ↓
输出：结构化、可执行的个性化就医方案
      ├── 初步病情判断与科室建议
      ├── 附近医院推荐（含距离、评级）
      ├── 挂号指引（科室、医生、挂号URL）
      ├── 出行路线规划（百度地图）
      └── 就医全程提示（携带物品、注意事项）
```

## 4. 核心 Skill 设计

### Skill 1：病情预判与科室推荐

**触发时机**：用户完成症状描述后立即调用

**输入**
```json
{
  "symptom_text": "最近头晕、失眠，有时头疼",
  "user_profile": { "age_group": "elderly" }
}
```

**执行逻辑**
1. 访问 `https://www.yixue.com/常见病自测/` 获取症状自测入口
2. 根据用户描述的主要症状定位对应页面（如 `https://www.yixue.com/常见病自测/常见症状辨病/头晕`）
3. 依据页面鉴别要点向用户补充询问（如：是否伴随耳鸣？起床时是否加重？）
4. GLM 综合症状信息，输出初步判断

**输出**
```json
{
  "preliminary_diagnosis": "可能为良性位置性眩晕或颈椎病",
  "recommended_departments": ["神经内科", "耳鼻喉科"],
  "warning_flags": ["若伴随言语不清请立即就急诊"],
  "need_more_info": false,
  "follow_up_questions": []
}
```

**关键约束**：输出须附加免责声明"仅供参考，不替代医生诊断"

---

### Skill 2：附近医院匹配

**触发时机**：用户确认科室建议后，提供所在地址时调用

**输入**
```json
{
  "user_location": "北京市朝阳区望京街道",
  "departments": ["神经内科"],
  "preferences": { "max_distance_km": 10, "hospital_level": "三甲" }
}
```

**执行逻辑**
1. 从 `data/医疗机构基本信息2023-03-29.csv` 加载北京医疗机构数据
2. 使用百度地图 MCP 计算用户位置到各医院的距离
3. 按距离过滤，筛选有目标科室的医院（科室信息从 yixue.com 补充）
4. 查询本地 `cache/blacklist.json`，过滤用户标记为厌恶的医院
5. 返回 Top-5 候选，附带距离、医院级别、科室信息

**输出**
```json
{
  "candidates": [
    {
      "hospital_name": "北京协和医院",
      "level": "三甲",
      "distance_km": 3.2,
      "departments": ["神经内科"],
      "address": "北京市东城区帅府园1号",
      "phone": "010-69156114",
      "yixue_url": "https://www.yixue.com/北京协和医院"
    }
  ],
  "data_source": "local_csv + yixue.com"
}
```

**交互**：向用户展示候选列表，等待用户选择目标医院

---

### Skill 3：挂号信息采集

**触发时机**：用户选定医院后调用

**输入**
```json
{
  "hospital_name": "北京协和医院",
  "department": "神经内科",
  "yixue_url": "https://www.yixue.com/北京协和医院"
}
```

**执行逻辑**
1. 访问 yixue.com 对应医院页面，提取官方网站 URL 及联系方式
2. 访问医院官网，定位挂号/预约入口（如"网上预约"菜单）
3. 采集挂号关键信息：
   - 网上挂号平台（京医通/医院自有系统）及直达 URL
   - 可预约时间段（如提前几天可约）
   - 挂号费用区间
   - 特需/普通门诊说明
4. 优先利用 `cache/hospital_info.json` 中缓存数据（避免重复爬取）

**输出**
```json
{
  "hospital_name": "北京协和医院",
  "registration_url": "https://www.bjguahao.gov.cn/...",
  "registration_platform": "京医通",
  "booking_note": "提前7天可预约，周一上午放号",
  "departments_available": ["神经内科普通号", "神经内科专家号"],
  "fee_range": "普通号9元，专家号300元+",
  "from_cache": false
}
```

**约束**：不执行自动预约，仅采集足够信息供用户自行挂号

---

### Skill 4：路线规划与就医行程单生成

**触发时机**：用户确认挂号信息（表明已挂号或即将挂号）后调用

**输入**
```json
{
  "user_location": "北京市朝阳区望京街道",
  "hospital_name": "北京协和医院",
  "hospital_address": "北京市东城区帅府园1号",
  "appointment_time": "2026-04-16 09:00",
  "output_format": "large_font_pdf",
  "user_profile": { "age_group": "elderly" }
}
```

**执行逻辑**
1. 调用百度地图 MCP 规划出行路线（驾车/公交/步行）
2. 计算预计出发时间（考虑提前到达、签到时间）
3. GLM 生成个性化提示（携带物品清单、院内导览步骤）
4. 生成 PDF 行程单（大字版/标准版）
5. 将本次就医信息写入 `cache/hospital_info.json` 持久化

**输出文档结构（大字版 PDF）**

```
🏥 最重要的就诊信息
   医院 / 科室 / 医生 / 时间

🎒 出门前请检查（清单）
   身份证 / 医保卡 / 病历 / 特殊提醒

🚗 怎么去最方便
   推荐打车：XX分钟 / 公交路线

🚶 到了医院怎么走
   分步骤文字导引

💡 就医提醒
   禁食要求 / 需空腹 / 穿着建议等
```

## 6. 本地持久化设计

**目录结构**

```
cache/
├── hospital_info.json       # 医院信息缓存（挂号URL、官网等）
├── user_history.json        # 用户历史就医记录（医院偏好）
├── blacklist.json           # 用户主动标记的厌恶医院
└── data_corrections.json    # 发现的数据错误及修正记录
```

**hospital_info.json 结构**
```json
{
  "北京协和医院": {
    "official_url": "https://www.pumch.cn/",
    "registration_url": "https://www.bjguahao.gov.cn/...",
    "registration_platform": "京医通",
    "last_verified": "2026-04-13",
    "departments": ["神经内科", "骨科", "..."]
  }
}
```

**data_corrections.json 结构**
```json
[
  {
    "source": "yixue.com/北京协和医院",
    "field": "phone",
    "wrong_value": "010-69156114",
    "correct_value": "010-69155114",
    "discovered_at": "2026-04-13",
    "verified": false
  }
]
```

**缓存策略**
- 医院基本信息（地址、电话）：7天过期
- 挂号 URL：24小时过期（号源变化频繁）
- 用户黑名单：永久保留，除非用户主动撤销

## 7. 错误处理与降级策略

| 异常情况 | 降级方案 |
|---------|---------|
| yixue.com 无法访问 | 使用本地 CSV 数据 + 提示用户自行搜索 |
| 医院官网结构变更，挂号 URL 提取失败 | 提供京医通/官网入口，提示用户手动找挂号入口 |
| 百度地图 MCP 调用失败 | 仅提供医院地址，提示用户使用手机地图导航 |
| 本地 CSV 中无目标医院 | 从 yixue.com 补充搜索，或提示用户修改筛选条件 |
| 信息不足无法判断科室 | 主动追问，最多询问 3 轮后给出保守建议 |

## 8. 典型场景示例

### 场景 A：老年人常规就医

**输入**：「老人头晕失眠两周了，帮我找附近能看的医院」

**流程**：
1. Skill 1 → 推荐神经内科/耳鼻喉科，询问是否伴随耳鸣
2. 用户回答 → 确认神经内科
3. Skill 2 → 展示附近 5 家三甲医院
4. 用户选择 → 某医院
5. Skill 3 → 提供京医通挂号直达链接，提示周一早放号
6. 用户告知已挂好号
7. Skill 4 → 生成**大字版 PDF**，含出行路线、携带清单、院内导引

---

### 场景 B：职场人夜间就医

**输入**：「我在望京上班，只能周末看颈椎，帮我找最近排队少的」

**流程**：
1. Skill 1 → 推荐骨科/推拿康复科
2. Skill 2 → 过滤有周末出诊的医院，优先距离近
3. Skill 3 → 提��周末专科门诊挂号入口
4. Skill 4 → 生成标准 PDF，含下班直达路线、时间损耗预估

---

### 场景 C：异地就医（扩展场景）

**输入**：「下周从赣州去广州看呼吸科，帮我把挂号和路线一起规划」

**流程**：
1. Skill 1 → 确认呼吸科
2. Skill 2 → 广州呼吸科强势医院推荐（如广医一院）
3. Skill 3 → 采集挂号信息
4. Skill 4 → 规划高铁+落地交通，生成三日医旅路书（含异地就医备案提醒）

## 10. 合规与免责

- 本系统仅提供**就医信息调度、流程辅助与行程规划**
- **不进行医疗诊断**，不替代执业医师判断
- 病情判断输出均附加免责声明
- 涉及用户位置等隐私信息，仅在当次会话中使用，不上传外部服务器
- 不执行自动挂号，仅提供挂号入口链接，敏感操作由用户本人完成
