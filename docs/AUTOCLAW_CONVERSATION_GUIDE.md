# 在AutoClaw中使用Zhishu Agent - 对话式指南

## 🎯 核心概念

AutoClaw是一个**对话式智能体框架**，用户通过自然语言与AutoClaw交互，AutoClaw理解意图后调用相应的Skills完成任务。

```
用户 (自然语言对话)
    ↓
AutoClaw (理解意图，调用Skills)
    ↓
Zhishu Agent Skills (执行任务)
    ↓
返回结果给用户
```

---

## 💬 使用方式 - 对话式交互

### 第一步: 注册Skills到AutoClaw

```bash
git clone https://github.com/jiezi1234/Zhishu.git
cd Zhishu
python config/autoclaw_integration.py
```

这会将Zhishu Agent的所有Skills注册到AutoClaw中。

---

### 第二步: 在AutoClaw中对话

用户只需要用**自然语言**描述就医需求，AutoClaw会自动调用Zhishu Agent来帮助。

#### **示例1: 老年人就医**

```
用户: "我奶奶最近腰疼得厉害，想在北京找个好医院看骨科，最好这周能挂上号，离家近一点最好。请帮我生成一份大字版的就医行程单。"

AutoClaw: 
[调用 healthpath-agent Skill]
[执行 Skill 1: 意图解析]
[执行 Skill 2: 号源搜索]
[执行 Skill 3: 方案决策]
[执行 Skill 4: PDF生成]

返回: "✅ 已为您生成就医行程单，文件已保存到: /output/appointment_itinerary_20260409_150839.pdf"
```

#### **示例2: 上班族就医**

```
用户: "我最近头晕，想找个离公司近的医院看神经内科，最好能在周末或晚上挂号。"

AutoClaw:
[调用 healthpath-agent Skill]
[自动执行完整流程]

返回: "✅ 已找到3个合适的医院方案，已生成PDF文件。"
```

#### **示例3: 异地就医**

```
用户: "我从外地来北京，需要看呼吸科，最好能找个评价好的医院，费用不要太贵。"

AutoClaw:
[调用 healthpath-agent Skill]
[自动执行完整流程]

返回: "✅ 已为您规划异地就医方案，包含交通、住宿等信息。"
```

## 🎯 AutoClaw会自动做的事情

当用户提出就医需求时，AutoClaw会：

1. ✅ **理解意图** - 提取科室、症状、时间、地点等信息
2. ✅ **搜索号源** - 从多个医院数据源搜索可用号源
3. ✅ **评分排序** - 基于距离、时间、费用等多个维度评分
4. ✅ **生成输出** - 生成美观的PDF就医行程单
5. ✅ **返回结果** - 告诉用户已完成，并提供PDF文件

---

## 💡 AutoClaw的智能特性

### 1. 自然语言理解
```
用户: "我想在北京找个好医院看骨科"
AutoClaw理解: 
  - 城市: 北京
  - 科室: 骨科
  - 任务: 搜索医院
```

### 2. 自动调用Skills
```
AutoClaw自动决定:
  - 调用 Skill 1 解析意图
  - 调用 Skill 2 搜索号源
  - 调用 Skill 3 评分排序
  - 调用 Skill 4 生成PDF
```

### 3. 错误处理
```
如果没有找到号源:
  AutoClaw会: "未找到匹配的号源，但已生成提示文档"

如果数据源不可用:
  AutoClaw会: "自动切换到备选数据源"

如果用户输入不清楚:
  AutoClaw会: "请问您想在哪个城市找医院？"
```

### 4. 上下文理解
```
用户: "我想看骨科"
AutoClaw: "好的，我来帮您找骨科医院。请问您在哪个城市？"

用户: "北京"
AutoClaw: "好的，在北京找骨科医院。您什么时候想看？"

用户: "这周"
AutoClaw: "明白了，为您搜索本周可挂号的骨科医院..."
```
